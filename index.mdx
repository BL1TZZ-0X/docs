Overview

Blitzz is a simple yet lightning-fast API for trading and market data on Solana tokens, powered by Pump Fun and Pump AMM. Use it to build transactions, send them, fetch token metadata, and stream live events.

Base URLs

HTTP Base URL: http://YOUR_HOST:80

WebSocket URL: ws://YOUR_HOST:8099/

<Info>
Replace `YOUR_HOST` with your server address in all examples.
</Info>

Authentication

No API key is required to use the Blitzz API. However, per-IP rate limits apply to each endpoint.

Conventions

Public Keys: All public keys are represented as base58 strings.

Amounts:

amount for buy or create is specified in lamports (1 SOL = 1,000,000,000 lamports).

amount for sell is specified in token base units (6 decimals).

Slippage: slippage_pct is a percentage (e.g., 0.5 means 0.5%).

Priority Fee: prioFee is the priority fee in SOL (e.g., 0.0005). Keep it between 0.0 and 0.5 SOL.

Pool: pool must be one of "pump", "pump-amm", or "auto".

Errors: Errors are returned in a standard JSON format: { "error" "message" }.

Local Trade Endpoint

Build an unsigned transaction that you can sign with your own wallet and send through your preferred RPC provider.

Method: POST

Path: /local-trade

Rate Limit: 5 requests/second per IP

Request Body
<ParamField body="type" type='"buy" | "sell" | "create" | "claimCreatorFees"' required>
The type of transaction to build.
</ParamField>
<ParamField body="pool" type='"pump" | "pump-amm" | "auto"' required>
The liquidity pool to use for the trade.
</ParamField>
<ParamField body="mint" type="string" required>
The base58 mint address of the token (required for `buy` and `sell`).
</ParamField>
<ParamField body="payer" type="string" required>
The base58 public key of the fee payer.
</ParamField>
<ParamField body="amount" type="number" required>
- For `buy`/`create`: The amount of SOL in lamports.
- For `sell`: The amount of tokens in base units (6 decimals).
</ParamField>
<ParamField body="slippage_pct" type="number" required>
The slippage tolerance as a percentage (e.g., `0.5` for 0.5%).
</ParamField>
<ParamField body="prioFee" type="number" required>
The priority fee in SOL (e.g., `0.0005`).
</ParamField>
<ParamField body="metadata" type="object">
Required for `type="create"`. Contains `name`, `symbol`, and `uri`.
</ParamField>
<ParamField body="transferConfig" type="object">
Optional configuration to transfer SOL in the same transaction. Contains `pubkey` (base58) and `lamports` (string).
</ParamField>

Responses
<ResponseExample>
```json 200 OK
{
"tx": "<base64-encoded-unsigned-transaction>"
}
```
</ResponseExample>

<ResponseExample>
```json 4xx/5xx Error
{
"error": "Descriptive error message"
}
```
</ResponseExample>

Blitzz Mode Endpoint

Build, sign, and send a transaction in a single API call. The API signs with the provided private key and waits up to 15 seconds for on-chain confirmation.

Method: POST

Path: /blitzz

Rate Limit: 3 requests/second per IP

Request Body

Includes all fields from /local-trade, plus:

<ParamField body="privateKey" type="string" required>
The base58 private key used to sign the transaction and pay fees.
</ParamField>

<Warning>
**Handle with care**: Never expose your private keys in client-side code or public repositories. For a fully self-custodial flow, use the `/local-trade` endpoint.
</Warning>

Responses
<ResponseExample>
```json 200 OK
{
"status": "success",
"signature": "<transaction_signature>"
}
```
</ResponseExample>

<ResponseExample>
```json 4xx/5xx Error
{
"error": "Descriptive error message"
}
```
</ResponseExample>

Example: Auto-Buy
<RequestExample>
```bash cURL
curl -X POST http://YOUR_HOST:80/blitzz \
-H 'content-type: application/json' \
-d '{
"type": "buy",
"pool": "auto",
"mint": "MINT_ADDRESS",
"amount": 10000000,
"slippage_pct": 0.5,
"prioFee": 0.0005,
"privateKey": "YOUR_PRIVATE_KEY"
}'
```
</RequestExample>

Metadata Endpoint

Fetch on-chain Metaplex metadata for one or more tokens.

Method: POST

Path: /metadata

Rate Limit: 1 request/second per IP

Limit: Up to 10 mints per request

Request Body
<ParamField body="mints" type="array<string>" required>
An array of 1 to 10 base58 mint addresses.
</ParamField>

Example Request
<RequestExample>
```bash cURL
curl -X POST http://YOUR_HOST:80/metadata \
-H 'content-type: application/json' \
-d '{
"mints": [
"MINT_ADDRESS_1",
"MINT_ADDRESS_2"
]
}'
```
</RequestExample>

Data WebSocket

Stream real-time events, including trades, token creations, and migrations.

URL: ws://YOUR_HOST:8099/

Path: /

Limits:

Max 2 connections per IP.

You must subscribe to at least one filter within 10 seconds of connecting.

Subscribing to Events

Send a JSON message to the WebSocket to subscribe to events. You can filter by mint, account, or event type.

<AccordionGroup>
<Accordion title="Subscribe to mints">
```json
{ "method": "subscribeFilterMint", "pubkeys": ["MINT1", "MINT2"] }
```
</Accordion>
<Accordion title="Subscribe to accounts (trader/creator)">
```json
{ "method": "subscribeFilterAccount", "pubkeys": ["ACCOUNT1"] }
```
</Accordion>
<Accordion title="Subscribe to new token creations">
```json
{ "method": "subscribeNewCreation" }
```
</Accordion>
<Accordion title="Subscribe to migration events">
```json
{ "method": "subscribeMigrations" }
```
</Accordion>
</AccordionGroup>

Errors and Limits
Common HTTP Errors

400 Bad Request: Invalid input (e.g., bad pubkey, amount <= 0).

404 Not Found: Item not found (e.g., token or pool does not exist).

429 Too Many Requests: Rate limit exceeded.

500 Internal Server Error / 502 Bad Gateway: Server or upstream error.

503 Service Unavailable: Blockhash is temporarily unavailable.

504 Gateway Timeout: Timed out waiting for transaction confirmation on /blitzz.

Best Practices
<Tip>
- **Slippage**: Keep slippage reasonable to avoid failed trades.
- **Pool Selection**:
- `"pump"`: For bonding-curve trades.
- `"pump-amm"`: For AMM trades.
- `"auto"`: Let the API decide the best pool.
- **Security**: For `/blitzz`, use a dedicated hot wallet with limited funds. For maximum security, use `/local-trade` to sign transactions locally.
</Tip>
